# truSDX-AI Driver - Quick Start Guide

## Prerequisites

- TruSDX QRP Transceiver connected via USB
- Linux system (Ubuntu, Mint, Debian)
- Python 3.6 or higher
- JS8Call or WSJT-X software

## Installation

1. **Install Dependencies:**
   
   Ensure you have all necessary dependencies installed:
   ```bash
   chmod +x setup.sh
   ./setup.sh
   ```

2. **Verify Installation:**
   
   Check that your TruSDX is detected:
   ```bash
   lsusb | grep -i trusdx
   ls /dev/ttyUSB*
   ```

## Running the Driver

3. **Start the Driver:**
   
   Start the truSDX AI-enhanced driver:
   ```bash
   python3 trusdx-txrx-AI.py
   ```
   
   For verbose output (recommended for first run):
   ```bash
   python3 trusdx-txrx-AI.py --verbose
   ```

## Software Configuration

4. **Configure JS8Call:**
   - Go to Settings → Radio
   - Set Radio to: **Kenwood TS-480**
   - Set CAT control to: **Network**
   - Host: `localhost`
   - Port: `4532`
   - PTT Method: **CAT**
   - Mode: **USB**

5. **Configure WSJT-X:**
   - Go to Settings → Radio
   - Set Radio to: **Kenwood TS-480**
   - Set CAT control to: **Network**
   - Host: `localhost`
   - Port: `4532`
   - PTT Method: **CAT**
   - Mode: **Data/PKT**

## Testing

6. **Test the Connection:**
   - Start JS8Call or WSJT-X
   - Verify frequency synchronization
   - Test TX/RX switching
   - Check VU meter activity

## Troubleshooting

- **No TruSDX device found**: Check USB connection and permissions
- **CAT control error**: Ensure driver is running and port 4532 is available
- **Audio issues**: Verify PulseAudio is running and devices are detected
- **Permission denied**: Add user to dialout group: `sudo usermod -a -G dialout $USER`
